# Дата: 2026-03-30

## Что было сделано
В субботу занимался реализацией методов для обновления данных в базе Supabase — чтобы сохранять прогресс и ответы пользователя при прохождении виджетов.\
В итоге было принято решение, что времени на полноценную интеграцию базы данных в приложение не хватает, поэтому на текущем этапе используем моковые данные с возможностью сохранения прогресса в localStorage.
В связи с этим переделал страницу библиотеки с использованием существующих моковых сервисов и исправил баг с кнопкой в Code Ordering Widget.
Также нужно откатить страницу аутентификации, чтобы она не использовала реализованный state, и оставить только стандартные методы Supabase для проверки, авторизован пользователь или нет.
Сейчас основной фокус — максимально упростить функционал, чтобы получить стабильную рабочую версию.
При этом есть возможность в течение недели отдельно доделать state и показать его на ревью как personal component, не затрагивая основной код проекта.
