Написала функции для работы с объектами в localstorage.  
Планируется такая архитектура:  
Для создания локальных объектов пользователя, его сессий и прогресса используются асинхронные API функции при логине. 
Далее вся логика выполняется локально с обращением к локальному хранилищу с помощью своей версии методов CRUD.  
При переходе на страницу практики так же создаётся объект текущей сессии.  
При работе с конкретным виджетом создаётся объект Answer, который нигде не сохраняется. После решения задачи он собирает результат и лобавляется в объект текущей сессии.  
При выходе из страницы практики объект текущей сессии добавляется в объект всех сессии юзера и обновляется прогресс.  
Для получегия виджетов по-прежднему обращаемся к mock/серверу.  
Для отрисовки дашборда и страницы профиля обращаемся к актуальным данным в localstorage.
