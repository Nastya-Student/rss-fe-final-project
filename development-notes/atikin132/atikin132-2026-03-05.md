Стратегии получились однообразные и выглядят вот так:
```typescript
export const quizStrategy: WidgetStrategy<QuizWidget,QuizAnswer> = {
  type: "quiz",
  
  render(widget, onAnswer) {
    return quizWidget(widget.payload, onAnswer);
  },

  validate(widget, answer) {
    return answer.answer === widget.payload.correctIndex;
  },
};

```

Так как стратегии уже были готовы нужно было теперь их применять. Так как в [примере](https://github.com/rolling-scopes-school/tasks/blob/master/stage2/tasks/rs-tandem/examples/01-widget-trainer/widget-engine.md#%D1%80%D0%B5%D1%88%D0%B5%D0%BD%D0%B8%D0%B5-strategy-pattern) было реализовано через функцию, я подумал, что наверное лучше создать класс с методами, чтобы всё было в одном месте и вызывалось через объект. Создал класс `WidgetEngine` и написал два метода `render()` и `handleAnswer()`.  В примере все стратегии помещались в `Map`, ну подумал, что сделаю так же. В итоге в `render()` передавал `widget: Widget`, брал из `Map` стратегию по ключу (типу widget). Затем обратился к стратегии и вызвал метод `render()` уже у неё, передал `widget`, но всё же не может "идти по маслу" ... .

Так как WidgetEngine рендерил один виджет за раз, то я решил, что теперь нужно браться за сессию обучения, чтобы рендерились все виджеты на тему.

Добавил поле `private widgets: Widget[];` и в конструкторе инициализирую его. 

Сделал так, чтобы когда пользователь нажимает кнопку "Submit", то происходит валидация, и показывается правильно он ответил или нет. И появляется кнопка "Next", при нажатии на которую текущий виджет удаляется, и показывается следующий. ## Решения (или попытки)

Пытался искать способы в интернете, но это не увенчалось успехом. Пошел за помощью к ИИ. Начал объяснять Chat GPT, что мне нужно, он делал вид, что понимал меня, но выдал полнейшую ерунду, то с `as`, то с `any`, то с `!`, а то всё разом. Не знаю с какой попытки, но в итоге получилось то, что сработало, а именно тип данных по типу `Map` с ключом и значением, где ключ это тип виджета (quiz, true-false, и т.д.), а значение это уже его тип (`QuizWidget`, `TrueFalseWidget`) и такой же `Map` для ответов, где ключ это тип виджета (quiz, true-false, и т.д.), а значение это уже его тип ответа (`QuizAnswer`, `TrueFalseAnswer`) . И в итоге отказался от `Map` для хранения стратегий и взял объект:
```typescript
const widgetStrategies: {
  [K in WidgetType]: WidgetStrategy<WidgetMap[K], WidgetAnswerMap[K]>;
} = {
  quiz: quizStrategy,
  "true-false": trueFalseStrategy,
  "code-completion": codeCompletionStrategy,
  "code-ordering": codeOrderingStrategy,
  "memory-game": memoryGameStrategy,
  "stack-builder": stackBuilderStrategy,
  "async-sorter": asyncSorterStrategy,
};
```
И в итоге получился такой `render()` у класса `WidgetEngine`:
```typescript
render<T extends WidgetType>(widget: WidgetMap[T] & { type: T }) {
    const strategy = widgetStrategies[widget.type];

    this.container.innerHTML = "";
    this.container.append(
      strategy.render(widget, (answer) =>
        this.handleAnswer(widget, strategy, answer),
      ),
    );
  }
```
В обработчике ответов тоже использовал дженерики:
```typescript
 private handleAnswer<T extends WidgetType>(
    widget: WidgetMap[T],
    strategy: WidgetStrategy<WidgetMap[T], WidgetAnswerMap[T]>,
    answer: WidgetAnswerMap[T],
  ) {
    new ParagraphCreator({
      parent: this.container,
      text: `Answer: ${JSON.stringify(answer)}`,
    });
    new ParagraphCreator({
      parent: this.container,
      text: `Correct: ${strategy.validate(widget, answer)}`,
    });
  }
```
Я рад, что в итоге что-то получилось, хоть и с помощью ИИ, но большую часть сегодняшнего времени ушло на эту проблему.
В итоге получился Widget Engine, но рендерит пока только один виджет за раз.
