# Дата: 2026-02-23

  

## Что было сделано

Начал разработку страницы dashboard. Так как есть [пример](https://github.com/rolling-scopes-school/tasks/blob/master/stage2/tasks/rs-tandem/examples/01-widget-trainer.md#dashboard), то решил что буду ориентироваться на него. Начал с приветствия пользователя и подумал, что уже сейчас не помешали бы моковые данные, но пока что оставил хардкодом. Затем начал разрабатывать компонент с прогрессом по темам, решил отображать общий прогресс и по каждой теме отдельно. И ещё раз убедился, что нужны данные, чтобы их отображать. 

Завел новую задачу по созданию моковых данных в Jira, назначил её себе.  

Так как нужно было подождать, пока команда проверит, что там получилось с моковыми данными. Я решил создать себе отдельно один интерфейс с прогрессом по темам, создал массив этих данных (пока что в коде создания элемента). 

Затем нужно было разработать утилиту для создания элемента `progress`. После этого, так как по всем темам будет одинаковый компонент прогресса, то создал progress-bar component. В функцию создания этого компонента передал название темы и проценты по ней. Когда создал элементы для названия темы, прогресса и процентов, то изначально они отображались не так как я хотел, а хотел я, чтобы всё названия тем были друг по другом, и аналогично с прогрессами и процентами. Чтобы этого достичь сначала создал отдельный контейнер для прогресса и процентов, затем использовал свойство шоркат `flex`, у названия темы и у процентов указал `flex: 0  0  25%;`, то есть это значит, что элементы должны занять по 25% родителя, до того, как свободное место будет распределено, а у прогресса указал `flex: 1;`, чтобы он занял всё сводоное место. То есть получилось так, что название темы занимает 25% от всего progress-bar component, потом добавил `gap: 10px;` и всё остальное место занимает контейнер с прогрессом и процентами. А уже в этом контейнере проценты занимают 25%, а всё остальное место сам прогресс. Задал процентам именно 25%, чтобы нормально отображалось, когда 1% и когда 100% и еще было пустое пространство до прогресса. 

Затем в функции создания progress component, где у меня уже был создан массив с данными, создал переменную `let  totalPercent  =  0;`, через цикл `for of` пробежался по массиву, и увеличивал переменную на процент по каждой теме, но чтобы два раза не бегать, здесь же создал progress-bar component для каждой темы и сразу добавил их в контейнер. Выглядит это так:
```typescript
for (const  topicProgress  of  topicProgressArr) {
	topicsProgressContainer.append(progressBarComponent(topicProgress.title, topicProgress.percent),);
	totalPercent  +=  topicProgress.percent;
}
```
Но так как, я хотел чтобы общий прогресс выглядел немного по-другому, то в функции создания progress-bar component я добавил такую проверку:
```typescript
if (progressTitle  ===  TOTAL_PROGRESS) {
	progressTopicContainer.classList.add("dashboard__total-progress");
	progressPercentContainer.classList.add("dashboard__total-progress");
	topicPercent.classList.add("dashboard__total-progress");
}
```
