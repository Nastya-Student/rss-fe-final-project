## Проблемы
А теперь собственно о проблеме, посмотрев в гугле и поспрашивая Chat GPT, который посоветовал использовать правило ```'header-pattern'``` ничего не сработало. Ругалось, что такого правила нет и кидало список актуальных правил. В итоге ещё поискав в гугле нашёл плагин [commitlint-plugin-regex-match](https://www.npmjs.com/package/commitlint-plugin-regex-match), но и он работать почему-то не захотел.


## Решения (или попытки)

В итоге поспрашивал ещё у Chat GPT как всё-таки можно сделать то, что я хотел. Он предложил написать кастомное правило, которое проверяло бы коммиты по структуре. В итоге он написал вот это:
```javascript
	"header-format": [2, "always"],
}, 

plugins: [
	{
		rules: {
			"header-format": ({ header }) => {
				const  regex  = /^(feat|fix|docs|style|refactor|test|chore|ci|build|perf|revert|init): RST-\d+ .+/;
				const  valid  =  regex.test(header);
				return [valid, "Commit must be in format: <type>: RST-<number> <description>",];
			},
		},
	},
],
```
Я проверил несколько раз и это действительно работает, правда не знаю, так нужно было это реализовать, или как-то по-другому, но проблема была решена.
