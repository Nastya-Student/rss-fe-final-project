## Что было сделано

Всё ещё болею, но решил, что нужно попробовать что-то сделать.

Так как дизайн Async Sorter Widget уже был готов в прошлый раз, сейчас нужно было сделать функционал. Решил начать с drag and drop. 

Затем добавил функцию `getAnswer()`, которая принимает массив с `dropZone`, куда пользователь перемещает блоки, проходится по нему и собирает массивы для `AsyncSorterAnswer`. В итоге получилась такая функция:
```typescript
function  getAnswer(dropZones:  HTMLElement[]):  AsyncSorterAnswer {
	const  answer: { name:  string; items:  string[] }[] = [];
	const  output:  string[] = [];
	for (const  dropZone  of  dropZones) {
		const  name  =  dropZone.dataset.name;
		if (name  !==  undefined) {
			const  items  = [...dropZone.children].map((el) =>  el.textContent);
			output.push(...items);
			answer.push({ name, items });
		}
	}
	answer.push({ name:  "output", items:  output });
	return { answer };
}
```
Затем изменил `validate` в `asyncSorterStrategy`, реньше там была заглушка, а теперь нормальная проверка
```typescript
validate(widget, answer) {
	const  correct  =  widget.payload.buckets;
	const  answerMap  =  new  Map(answer.answer.map((bucket) => [bucket.name, bucket.items]));
	return  correct.every((bucket) => {
		const  answerItems  =  answerMap.get(bucket.name);
		if (!answerItems) {
			return  false;
		}
		if (answerItems.length  !==  bucket.correctItems.length) {
			return  false;
		}
		return  bucket.correctItems.every((item:  string, index:  number) =>  item  ===  answerItems[index],);
	});
},
```

Потом добавил функционал для кнопки `Run`. При нажатии на неё происходит следующее: очищается `outputContainer`, вызывается `getAnswer()` и берется из него не весь `AsyncSorterAnswer`, а только `output`. И с задержкой 0,4 c элементы появляются в `outputContainer`.

Переписал с использованием констант. И сделал [Pull Request #34](https://github.com/Nastya-Student/rss-fe-final-project/pull/34)

## Решения (или попытки)

Начал читать как вообще делают drag and drop, пробовал то же самое, ничего не получилось. Хорошо, подумал я, если стандартный HTML drag and drop не работает, то попробую использовать какую-нибудь библиотеку. Начал искать какие вообще есть, нашел несколько библиотек, но решил остановиться на Dragula, потому что понравилось название. 

Попробовал подключить Dragula, пришлось изменять tsconfig.json, начал использовать библиотеку, но полезли ошибки TypeScript, ему не понравилось отсутствие типизации. Понял, что нужно как-то попытаться типизировать, но подумал, что раз с этой библиотекой не клеится, попробую другую.

В этот раз остановился на SortableJS, попробовал использовать, с этой библиотекой таких проблем не возникло, но результат оказался таким же, как с HTML drag and drop, ничего не заработало. Элементы можно тянуть, и они идут за мышкой, но когда отпускаешь над контейнером, элемент возвращаются обратно.

Я расстроился, что ничего не выходит. 

Решил вернуться обратно к HTML drag and drop. Начал по одному добавлять слушатели ивентов. Начал с `dragstart` вывел в консоль элемент, который хочу переместить, он появился. `dragend` также работал корректно. Далее повесил слушатели уже на блоки в которые мне нужно перемещать, а именно `dropZone`, а слушатель `dragover`. Попробовал вывести в консоль блок, в который хочу переместить элемент, и вот здесь уже ничего не появилось в консоли. Я начал водить этим несчастным блоком по всему экрану, и в какой-то момент заметил в консоли изменения. Я нашел место на которое реагирует `dragover`, но оно было далеко от того, куда бы я хотел поместить свой блок, хотя в консоли выдавало именно нужное место. В итоге не знаю, сколько с этим провозился, может 30 минут, а может и час, но я нашел вариант, при котором всё работает как ожидается. Обычно, когда я что-то разрабатываю у меня в DevTools стоит режим с device toolbar и 200% масштаб. Я попробовал отключить этот режим, и всё заработал корректно, но когда снова переключался в него снова были те же проблемы. Я решил делать дальше, а с этим как-нибудь попозже разобраться.

В итоге сделал drag and drop с использоваем HTML drag and drop, сделал так, чтобы всё корректно работало и отображалось. Но работало оно только в режиме без device toolbar, при переключении на него всё ломалось. Но не только это меня смущало, а еще то, что на мобилке оно вообще не работало, даже просто перемещаться не хотело. Я поискал в интернете, и оказалось, что HTML drag and drop достаточно плохо работает с мобильными устройствами, а вот SortableJS, от которого я давно отказался, нормально работает с мобилками.

В итоге решил попробовать вернуться на SortableJS и попробовать в режиме без device toolbar, и всё заработало. И при включении device toolbar, если включить мобильные устройства, то оно тоже перетаскивалось. 

И в какой-то момент времени в режиме с device toolbar, я попробовал переключить масштаб c 200% на 100%. И оно всё начало работать, как ожидается. Я обрадовался и решил остаться на реализации с SortableJS.
